# Greybird + Stylix Integration

This document explains how the Greybird GTK theme is integrated with Stylix to provide wallpaper-based color theming.

## Overview

The configuration uses a forked version of [Greybird](https://github.com/breitnw/Greybird) that supports custom accent colors. Instead of using fixed color schemes, the theme is built dynamically using colors generated by Stylix from the current wallpaper.

## How It Works

### Color Generation

1. **Stylix** analyzes the wallpaper image (`home-manager/wallpapers/still_wallpapers/wallhaven-1p5z29.jpg`)
2. A genetic algorithm generates a base16 color palette from the image
3. The palette is accessible via `config.lib.stylix.colors` with slots `base00` through `base0F`

### Theme Building

The `home-manager/theming/greybird.nix` file creates a derivation that:

1. Fetches the Greybird source from `github:breitnw/Greybird/master`
2. Renames the theme to `greybird-stylix-{base00}` to prevent conflicts
3. Replaces hardcoded colors in GTK3 SCSS files with Stylix-generated colors
4. Replaces hardcoded colors in GTK2 gtkrc files
5. Builds the theme using meson

### Color Mapping

The base16 palette maps to Greybird colors as follows:

| Base16 Slot | Greybird Usage |
|-------------|----------------|
| `base01` | Background, base colors |
| `base02` | Panel background |
| `base05` | Foreground, text |
| `base06` | Selected text, white |
| `base0E` | Accent color (selected_bg_color) |

## File Structure

```
home-manager/theming/
├── default.nix          # Imports all theming modules
├── stylix.nix           # Stylix configuration (colors, fonts, targets)
├── greybird.nix         # Greybird theme derivation
├── font-settings.nix    # Custom font options
└── hm-ricing-module.nix # hm-ricing-mode integration
```

## Configuration

### Changing the Wallpaper

To change the wallpaper and regenerate colors:

1. Update the `image` path in `home-manager/theming/stylix.nix`:
   ```nix
   stylix = {
     image = ../wallpapers/still_wallpapers/your-new-wallpaper.jpg;
   };
   ```

2. Rebuild home-manager:
   ```bash
   home-manager switch --flake ~/GitRepos/nixos-dotfiles#railgun-linux-desktop
   ```

### Using a Fixed Color Scheme

To use a fixed color scheme instead of wallpaper-based generation:

1. Uncomment the `base16Scheme` line in `stylix.nix`:
   ```nix
   base16Scheme = "${pkgs.base16-schemes}/share/themes/atelier-forest.yaml";
   ```

2. Comment out or remove the `image` line

3. Rebuild home-manager

### Customizing the Theme

The Greybird theme can be customized by modifying the sed replacements in `greybird.nix`. The current mapping follows the pattern established by the upstream fork.

## Dependencies

- **Stylix**: Generates base16 colors from wallpaper
- **Greybird fork**: `github:breitnw/Greybird/master` - GTK theme with custom accent support
- **GTK engine**: `gtk-engine-murrine` - Required for GTK2 theme rendering

## Troubleshooting

### Theme Not Applying

1. Verify the theme is built:
   ```bash
   ls ~/.themes/greybird-stylix-*
   ```

2. Check GTK configuration:
   ```bash
   gsettings get org.gnome.desktop.interface gtk-theme
   ```

### Colors Look Wrong

1. Check the generated palette:
   ```bash
   home-manager generations | head -1
   ```

2. Verify Stylix is generating colors from the wallpaper

### Build Errors

If the build fails, check:

1. The Greybird source is accessible
2. All sed replacements are valid
3. The meson build completes successfully
